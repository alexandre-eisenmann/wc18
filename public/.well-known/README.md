# `.well-known/` — Apple Pay domain verification

This directory is served at `https://<domain>/.well-known/` and is used by
Apple Pay to verify that we control the domain.

## What goes here

Exactly one file:

    apple-developer-merchantid-domain-association

(no extension). The file contents are generated by Stripe and are unique per
domain. You cannot make this file up — it must come from the Stripe Dashboard.

## How to add it

1. In the Stripe Dashboard (account owner, **Live mode**), open
   **Settings → Payment methods → Apple Pay** and click **Add a new domain**.
2. Enter the live domain (e.g. `bolaodosboloes.com.br` — not the
   `*.web.app` Firebase preview).
3. Stripe shows a `Download` link for the verification file. Download it.
4. Place the downloaded file in this folder, named exactly
   `apple-developer-merchantid-domain-association` (no `.txt`, no extension).
5. Vite copies `public/` → `dist/` at build time, and Firebase Hosting serves
   it as a static file (`firebase.json` no longer ignores `.well-known/`).
6. Deploy:

   ```bash
   pnpm build
   firebase deploy --only hosting
   ```

7. Back in the Stripe Dashboard, click **Verify**. Stripe will fetch
   `https://<domain>/.well-known/apple-developer-merchantid-domain-association`
   and confirm the contents match.

## Notes

- Google Pay does NOT need any of this — it works as soon as `card` is
  enabled on the Stripe account, with no domain verification step.
- The verification file should be re-downloaded if the Apple Pay merchant ID
  on the Stripe account changes (rare).
- This README itself is harmless to deploy — Apple only looks for the
  specific filename above.
